单Target多Configuration实现分环境SDK依赖及分环境Host配置

前景

公司APP在Debug环境和Release环境下需要使用两个不同的三方SDk，但这两个SDK暴露的API是相同的（虽然搞不懂为什么会这么奇葩，但事实却是如此），最开始的解决方案是新建了两个Target：

  • ReleaseTarget配置正线Host及正线三方SDK
  • DebugTarget配置测试Host及测试SDK

不同环境的判断也是通过两个Target配置的不同CFBundleDisplayName来判断的。

两个Target来解决以上问题会多出一些额外的工作量比如：

  • 新建代码文件或者资源文件时需要勾选两个Target
  • 修改一些Target配置项时也需要同时修改两个Target（集成一个三方SDK配置其需要的BuildSetting）
  • 添加一些RunScript时也需要同时修改两个Target

灵感

要解决上面的问题又不会在之后增加额外的工作量，我立即想到的是之前使用Cocoapods来管理三方库时可以配置的Configuration来选择性依赖一些三方库比如：FLEXTool、DoraemonKit等

  • Debug Config对应的包（配置测试环境Host和测试ThirdSDK，用于开发调试及测试同学测试）
  • Release Config对应的包（配置正式环境Host和正式环境ThirdSDK，用于上线）
  • Debug_For_Release Config对应的包（配置正线环境Host和正式环境的ThirdSDK，用户开发调试及测试同学测试），且该配置项复制与Debug对应的配置项，如何新增Configuration及了解其概念请自行上网搜索
