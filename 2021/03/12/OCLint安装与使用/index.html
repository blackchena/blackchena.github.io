错误一：注意如果不加export LC_CTYPE=en_US.UTF-8会报这样的错误：

/Library/Ruby/Gems/2.6.0/gems/xcpretty-0.3.0/lib/xcpretty/parser.rb:434:in ===': invalid byte sequence in US-ASCII (ArgumentError)

错误二：遇到了这样的错误：

xcpretty: command not found

  • 解决步骤：

    • 在Xcode Script中打印PATH发现其中没有xcpretty所在的bin路径，但是在Terminal打印是有该bin路径的

    • 在Script第一行加入export PATH="$PATH:~/.rvm/gems/ruby-2.5.1/bin"

      报错：ruby_executable_hooks: No such file or directory

    • 把上述路径换成export PATH="$PATH:$HOME/.rvm/gems/ruby-2.5.1/bin"

      报错： /System/Library/Frameworks/Ruby.framework/Versions/2.6/usr/lib/ruby/2.6.0/rubygems.rb:283:in find_spec_for_exe': can't find gem xcpretty (>= 0.a) with executable xcpretty (Gem::GemNotFoundException)

    • 由于我的Mac安装了rvm环境，最终发现我rvm用的ruby版本和Mac自带的ruby版本有差异，而且Xcode用的是Mac自带的ruby版本，我这里想了两种解决方法：

      • 因为Xcode用的Mac自带ruby版本，rvm use system后重新安装xcpretty，同时删除上面配置环境变量PATH那一行（成功）
      • 在Run Script中第一行加入rvm use ruby-2.5.1，这里用的ruby版本号根据自己的xcpretty装到哪个版本的ruby环境来设置（报错rvm: command not found 看来在Run Script中设置ruby 版本没有作用）
