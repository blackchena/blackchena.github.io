Flutter与iOS原生通信方式

网上普遍有两种教程

  1. 是在Appdelegate里面写相应的插件逻辑

  2. 通过Android Studio创建一个Flutter Plugin的项目，在生成的插件类中写插件逻辑

    GeneratedPluginRegistrant.h

    GeneratedPluginRegistrant.m

    AudioRecorder2Plugin.m

    SwiftAudioRecorder2Plugin.swift audioRecorder2/SwiftAudioRecorder2Plugin.swift at master · ApurvH/audioRecorder2

以上两种方式都初始化了FlutterMethodChannel，以及注册了收到flutter消息时Native回调方式

  • Block回调方式
    let batteryChannel = FlutterMethodChannel(name: "audio_recorder", binaryMessenger: registrar.messenger())
    `batteryChannel.setMethodCallHandler({
    (call: FlutterMethodCall, result: @escaping FlutterResult) -> Void in
// Note: this method is invoked on the UI thread.
// Handle battery messages.
    })`
  • Delegate回调方式
    `let channel = FlutterMethodChannel(name: “samples.flutter.dev/battery”,
                                                binaryMessenger: controller.binaryMessenger)`
    let instance = SwiftAudioRecorder2Plugin()
    registrar.addMethodCallDelegate(instance, channel: channel)
